
1. Definição do Escopo do MVP (O Que Construir)
✅ DENTRO DO ESCOPO
Auth-01: Cadastro de Usuário

Justificativa: Base para identificar clientes e persistir dados.
Escopo: Nome, email, senha, CPF, data de nascimento.

Auth-02: Login de Usuário

Justificativa: Acesso à conta e histórico.
Escopo: Email + senha.

Produto-01: Listagem de Produtos

Justificativa: Core da aplicação - catálogo de medicamentos/suplementos.
Escopo: Lista paginada com foto, nome, preço, estoque.

Produto-02: Detalhes do Produto

Justificativa: Informações para decisão de compra.
Escopo: Descrição, preço, composição, instruções de uso.

Produto-03: Busca de Produtos

Justificativa: Facilitar encontrar produtos específicos.
Escopo: Busca por nome (texto simples).

Carrinho-01: Adicionar ao Carrinho

Justificativa: Essencial para fluxo de compra.
Escopo: Adicionar/remover produtos, ajustar quantidade.

Carrinho-02: Visualizar Carrinho

Justificativa: Revisar antes de finalizar.
Escopo: Lista de itens, subtotal.

Pedido-01: Finalizar Pedido (Simplificado)

Justificativa: Converter carrinho em pedido.
Escopo: Criar pedido com status "pendente", sem pagamento real (simulado).

Perfil-01: Visualizar Perfil

Justificativa: Usuário precisa ver/editar dados básicos.
Escopo: Nome, email (edição mínima).

2. Fluxos Críticos do MVP (Como Usar)
Fluxo 1: Primeiro Acesso e Compra
1. Usuário abre o app
2. Usuário clica em "Criar Conta"
3. Usuário preenche formulário (nome, email, senha, CPF, data nascimento)
4. Informa dados de endereço
5. Informa dados de pagamento
6. Sistema cria conta e autentica automaticamente
7. Usuário navega pela listagem de produtos
8. Usuário clica em um produto → vê detalhes
9. Usuário adiciona ao carrinho (define quantidade)
10. Usuário repete passos 5-7 para outros produtos (opcional)
11. Usuário clica no ícone do carrinho
12. Usuário revisa itens e clica "Finalizar Pedido"
13. Sistema cria pedido e exibe confirmação
```

### Fluxo 2: Usuário Recorrente
```
1. Usuário abre o app
2. Usuário insere email e senha → Login
3. Usuário busca produto específico (barra de busca)
4. Usuário adiciona ao carrinho
5. Usuário finaliza pedido (igual fluxo 1, passos 9-11)
```

### Fluxo 3: Gestão de Perfil
```
1. Usuário logado clica em "Perfil"
2. Sistema exibe dados cadastrais
3. Usuário edita nome (opcional)
4. Sistema salva alterações

3. Mapeamento de Serviços/Módulos e Dependências
Backend (Supabase)
Serviço A: Autenticação (Supabase Auth)

Descrição: Cadastro e login de usuários.
Dependências: Nenhuma (Base).
Entregáveis: Configuração Supabase Auth, funções de registro/login.

Serviço B: Tabela usuario

Descrição: Dados complementares do perfil (CPF, data_nascimento).
Dependências: Serviço A (chave estrangeira para auth.users).
Entregáveis: Schema SQL, RLS policies.

Serviço C: Tabela produto

Descrição: Catálogo de produtos.
Dependências: Nenhuma (pode ser populada com dados fake).
Entregáveis: Schema SQL, seed de 20-30 produtos fictícios.

Serviço D: Tabela carrinho

Descrição: Persistir itens do carrinho por usuário.
Dependências: Serviço A (usuario_id), Serviço C (produto_id).
Entregáveis: Schema SQL, RLS policies para isolamento por usuário.

Serviço E: Tabela pedido e item_pedido

Descrição: Registro de pedidos finalizados.
Dependências: Serviço A (usuario_id), Serviço C (produto_id).
Entregáveis: Schema SQL simplificado (sem pagamento real).

Frontend (React Native)
Módulo F1: Navegação e Estrutura

Descrição: Stack Navigator (React Navigation).
Dependências: Nenhuma (Base).
Entregáveis: Configuração de rotas, tab navigator.

Módulo F2: Telas de Autenticação

Descrição: Login e Cadastro.
Dependências: Módulo F1, Serviço A.
Entregáveis: 2 telas funcionais.

Módulo F3: Tela de Listagem de Produtos

Descrição: FlatList de produtos com imagem, nome, preço.
Dependências: Módulo F1, Serviço C.
Entregáveis: 1 tela com busca integrada.

Módulo F4: Tela de Detalhes do Produto

Descrição: Exibe produto individual + botão "Adicionar ao Carrinho".
Dependências: Módulo F1, Serviço C, Serviço D.
Entregáveis: 1 tela.

Módulo F5: Tela de Carrinho

Descrição: Lista itens, subtotal, botão "Finalizar Pedido".
Dependências: Módulo F1, Serviço D, Serviço E.
Entregáveis: 1 tela.

Módulo F6: Tela de Perfil

Descrição: Exibe e edita dados do usuário.
Dependências: Módulo F1, Serviço A, Serviço B.
Entregáveis: 1 tela.
